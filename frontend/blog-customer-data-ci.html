Project Overview

A Python data cleaning pipeline with full CI/CD automation using GitHub Actions. The project cleans messy customer data (names & emails) by trimming whitespace, lowercasing, removing nulls/empties, and deduplicating.

Repository: github.com/iimgaurav/customer_data_ci

What Was Built

1. Core Logic ‚Äî transform.py

The clean_dataframe() function processes customer data with the following transformations:

Step What it does
Trim names " John Doe " ‚Üí "john doe"
Trim emails " JOHN@EXAMPLE.COM " ‚Üí "john@example.com"
Remove nulls Drops rows with None/NaN emails
Remove empties Drops rows with "" emails
Deduplicate Keeps first occurrence per email
Reset index Clean sequential index

2. Tests ‚Äî test_transform.py

15 comprehensive unit tests covering all edge cases:

  • Trim/lowercase names and emails
  • Null, NaN, and empty email removal
  • Duplicate detection (case-insensitive, with spaces)
  • Index reset after cleaning
  • Original DataFrame immutability
  • Empty DataFrame handling
  • Complex multi-issue scenario

3. CI Pipeline ‚Äî ci.yml

Triggers on push/PR to main, master, develop with the following workflow:

  • Checkout repository
  • Setup Python environment
  • Install dependencies
  • Lint with flake8
  • Run tests with coverage
  • Display results

Key Features:

  • Matrix testing across Python 3.11, 3.12, 3.13
  • Flake8 linting with custom ignores
  • Test coverage reporting via pytest-cov
  • pip caching for faster runs

4. CD Pipeline ‚Äî cd.yml

Triggers on push to main/master with three deployment stages:

Job What it does
build_and_test Lint ‚Üí Test with coverage ‚Üí Zip src/ ‚Üí Upload artifact
deploy_dev Downloads artifact ‚Üí Simulates DEV deployment
deploy_prod Downloads artifact ‚Üí Simulates PROD deployment (requires approval)
